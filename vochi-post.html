3 thứ mình rút ra khi để Claude làm phần lớn
1

CLAUDE.md thay đổi tất cả

Mình viết một file CLAUDE.md khá kỹ mô tả kiến trúc, quy ước IPC, mấy luật không được phạm. Từ lúc có file này, agent gần như không còn tự dẫm chân ở những lần sửa sau — đây là thứ thay đổi nhiều nhất so với lúc đầu vibe code chay.

2

Frontend ngọt, native thì phải dẫn

Claude xử lý frontend và logic game rất ngọt, nhưng tới khúc native AppKit trên macOS (overlay trong suốt always-on-top sống sót khi app khác fullscreen, nhận phím khi cửa sổ không focus) thì phải đọc tài liệu cùng nó và chỉ hướng — để tự thì hay đoán mò.

3

Verify bằng output thật, không tin lời

Mình bắt nó chạy test và đọc output thật trước khi tin là xong, thay vì tin lời "đã fix". Cái này cắt được kha khá bug ảo.
